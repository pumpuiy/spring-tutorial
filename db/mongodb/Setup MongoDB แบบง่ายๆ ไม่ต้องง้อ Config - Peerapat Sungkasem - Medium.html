Setup MongoDB แบบง่ายๆ ไม่ต้องง้อ Config

Peerapat Sungkasem
Mar 22, 2017 · 2 min read

ใครที่เคย setup MongoDB แบบติดตั้งเองน่าจะเคยทำตามขั้นตอนคร่าวๆดังนี้

  1. install mongodb ผ่าน yum, apt-get หรือ setup file
  2. start mongodb service
  3. connect mongo shell เพื่อสร้าง user
  4. แก้ไข config file เปลี่ยน port เปิด auth
  5. restart service

ซึ่งแต่ละขั้นตอนก็มีรายละเอียดปลีกย่อยลงไปอีก วิธีการติดตั้งก็ขึ้นอยู่กับ OS ที่เราใช้ แถมการเข้าไปแก้ไข config file ของ mongo ก็ดูเป็นเรื่องน่าตื่นเต้นสำหรับมือใหม่ วันนี้เราจะมาลองหาวิธีที่ง่ายกว่านี้กัน

โดยความต้องการยังคงเดิมคือ

  • เปิดใช้งาน auth ด้วย username / password
  • ใช้ custom port แทน default port

สำหรับพระเอกของงานนี้ก็คือ Docker นั่นเอง developer คนไหนยังใช้ไม่เป็นต้องรีบศึกษาได้แล้วนะครับก่อนจะตกรถ

สร้าง container ของ MongoDB

อันดับแรกต้อง install docker ในเครื่องก่อนนะครับ สามารถ download และศึกษาวิธีการ install ได้จาก website ของ docker เลย (ตัวอย่างนี้ผมใช้ Docker for Mac)

จากนั้นก็ต้องหา image ของ mongodb มาให้ได้ก่อน ซึ่งพบว่ามี official image อยู่แล้วใน docker store พร้อมอธิบายวิธีใช้งานไว้เสร็จสรรพ

หลังจากอ่านวิธีใช้จนเข้าใจแล้ว (หรือไม่เคยอ่านกัน) ก็เริ่มกันเลยด้วยคำสั่ง

$ docker run --name mongodb -v /docker/data/db:/data/db -p 2277:27017 -d mongo:3.2.12 --auth

เสร็จแล้วครับ! ตอนนี้เราได้ mongodb ที่รันบน port 2277 พร้อมเปิด auth เรียบร้อย ลองทดสอบดูด้วยคำสั่ง docker ps จะเห็นว่ามี container mongodb รันอยู่

ทีนี้มาดูคำสั่ง docker กันซักหน่อยว่ามันทำอะไรบ้าง

docker run

เป็นคำสั่งที่ใช้รัน container ขึ้นมาจาก image ที่เราระบุ

--name mongodb

ระบุชื่อ container ที่เราจะรันขึ้นมาในที่นี้ตั้งชื่อว่า mongodb

-v /docker/data/db:/data/db

ทำการ map volume เพื่อเก็บ data ของ MongoDB ไว้ข้างนอก container ที่ path /docker/data/db

-p 2277:27017

ทำการ map host port 2277 ให้วิ่งไปที่ container port 27017 เพื่อให้ access จากข้างนอกเข้ามาที่ port 2277 แทน default port

-d

ทำการรัน container แบบ detached (รันเป็น background)

mongo:3.2.12

ระบุชื่อ image และ version

--auth

ระบุว่าให้เปิดใช้งาน auth บน mongodb

สร้าง user สำหรับ login database

เนื่องจากเรายังไม่มี user บน db เลย เราจึงต้องสร้าง user คนแรกขึ้นมาก่อน (ไม่งั้นจะเปิด auth เพื่อ?) โดย connect เข้าไปที่ mongo shell ของ container ด้วยคำสั่ง

docker exec -it mongodb mongo admin

จากนั้นก็สร้าง user ด้วย mongo shell

db.createUser({
  user: "admin", 
  pwd: "secure", 
  roles: [ { role: "root", db: "admin" } ]
})

เท่านี้ MongoDB ของเราก็พร้อมใช้งานแล้วครับ

จะเห็นว่าหลังจากสร้าง user คนแรกแล้ว ตัว auth จะทำงานทันที โดยเราจะไม่สามารถรัน command อะไรได้อีก ทดสอบด้วยคำสั่ง show dbs จะขึ้น error แบบนี้
